# Dependencies for the Vision OS demo.
#
# Why these versions:
#   opencv-python >= 4.10   -- needed for cv2.getWindowImageRect on macOS
#                              and for the WND_PROP_FULLSCREEN flag to
#                              actually take effect when paired with
#                              WINDOW_NORMAL.  Older 4.5.x wheels are
#                              buggy on Apple Silicon.
#   numpy        >= 1.26    -- last numpy 1.x with arm64 wheels; numpy 2.x
#                              also fine but cv2 wheels need to be matched.
#   Pillow       >= 10.0    -- font.set_variation_by_name landed in 9.5;
#                              10+ gives us the modern getbbox/getlength
#                              text-metrics API we rely on for layout.
#   mediapipe    >= 0.10    -- Phase 8 adds optional MediaPipe Hands.  If
#                              mediapipe is not installed the demo still
#                              runs in mouse-only mode (phase8_hand.py
#                              catches the import / camera failure and
#                              falls back), so this stays installable as
#                              an optional add-on from a runtime
#                              perspective; we pin a version known to work
#                              with the rest of the stack.  The 0.10+
#                              wheels on Apple Silicon Pythons support
#                              both numpy 1.x and 2.x at recent versions,
#                              so the numpy upper bound above stays open.
#
# We deliberately do NOT add scikit-learn, PyTorch, or pyautogui:
#   - scikit-learn comes back when the eye-tracking Part 2 phases land.
#   - PyTorch lives only behind the optional Phase 12 `--pretrained` flag.
#   - pyautogui is not needed.  Phase 8 keeps every gesture INSIDE the
#     fake OS canvas; we never drive the real macOS mouse, so the
#     cross-platform mouse-control story can be deferred indefinitely.

opencv-python>=4.10,<5
numpy>=1.26,<3
Pillow>=10.0,<12
# Pinned to 0.10.14 because mediapipe >= 0.10.30ish removed the legacy
# `mp.solutions.hands` / `mp.solutions.face_mesh` API in favour of the
# new `mp.tasks` API.  `src/hands.py` uses the legacy API, so we hold
# the line at the last release that still ships it.  Same pin the
# Hand-controller reference project uses.
mediapipe==0.10.14
